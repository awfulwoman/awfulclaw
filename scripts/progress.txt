## Codebase Patterns
- Telegram JSON responses are untyped dicts (`dict[str, object]`); pyright strict mode requires `str(v)` or similar casts rather than direct `int()` on `object` values
- The `anthropic` SDK is an optional dependency (extras = `vision`); import it inside the function that needs it with a helpful error on `ImportError`
- Pre-existing ruff E501 violations exist in `tests/` — only check `awfulclaw/` for clean lint on new code
