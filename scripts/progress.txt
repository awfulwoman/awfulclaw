## Codebase Patterns
- Module-level functions in `imessage.py` are preserved alongside the class — the class wraps them
- `uv run ruff check --fix` handles import sorting automatically
- Pre-existing lint errors (config.py E501, test_config.py I001) existed before this milestone; fix them when encountered
- Tools (ruff, pyright, pytest) must be invoked via `uv run --with <tool> <tool> ...` — they are dev deps not on PATH directly
- Connector classes live in their own module (e.g. `telegram.py`) and implement `Connector` from `connector.py`
- Skill tags (`<skill:imap/>`) follow the same intercept-strip-inject pattern as `<memory:write>` in `loop.py`
- `config.get_connector()` uses lazy imports to avoid loading connector deps unless needed
