## Codebase Patterns
- SQLite DB lives at `memory/awfulclaw.db`; always use `get_db()` from `awfulclaw/db.py` — it sets WAL mode and row_factory
- `init_db()` creates tables if not exists; call before first DB access
- Schedule tests use `monkeypatch.chdir(tmp_path)` to isolate the DB/files — `get_db()` uses relative `Path("memory/awfulclaw.db")` so this works transparently
- Pre-existing ruff errors in `test_telegram.py` and `test_startup_briefing.py`; pre-existing pyright errors in `claude.py`, `loop.py` (msg unknown type), and test files — don't count these as new failures
- `loop.run()` is async; entry point uses `asyncio.run(loop.run(connector))`; Claude calls use `run_in_executor` + `asyncio.Semaphore`
- `_dispatch_all_skills` is async; pass `sem` and `ev_loop` as params; closures inside `run()` can reference these directly
