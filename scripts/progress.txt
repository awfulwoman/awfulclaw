## [2026-03-30] - #95 Gateway abstraction: decouple message ingestion from agent loop
- Created `awfulclaw/gateway.py` with `Gateway` class: owns `dict[str, Connector]`, `queue.Queue[Message]` inbound queue, `threading.Event` stop signal, and background polling threads per connector
- Added `channel: str = "telegram"` to `Message` dataclass (default preserves backward compat)
- Added `get_channel()` to `config.py` so `__main__.py` can name the channel when building the gateway
- Changed `loop.run(connector)` → `loop.run(gateway)`: drains `gateway.get_messages()` each tick; sends via `gateway.send(channel, recipient, body)`; calls `gateway.start()` on entry and `gateway.stop()` on exit
- Updated `__main__.py` to create `Gateway([(channel, connector)])` before calling `loop.run()`
- Files changed: `awfulclaw/gateway.py` (new), `awfulclaw/connector.py`, `awfulclaw/config.py`, `awfulclaw/loop.py`, `awfulclaw/__main__.py`
- **Learnings for future iterations:**
  - `queue.Queue` (threading) is the right choice here — loop is synchronous, asyncio would require a full rewrite
  - Gateway background threads use `threading.Event.wait(timeout)` for the poll interval — avoids busy-loop and responds to `stop()` promptly
  - `primary_recipient_for(channel)` method lets message-handler replies route back to the correct channel without exposing `_connectors` dict
  - Pre-existing pyright errors (67) and ruff errors (2) in test_telegram.py and startup_briefing — don't touch those files
  - The `now = datetime.now(timezone.utc)` line in the idle block was unused after the gateway removed `last_poll` tracking — removed it to avoid pyright warnings
