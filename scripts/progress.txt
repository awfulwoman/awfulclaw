## [2026-03-29] - #65 Add created_at anchor to Schedule model for drift-free next-run computation
- Added `created_at: datetime` to `Schedule` dataclass; set in `Schedule.create()` to `datetime.now(timezone.utc)`
- `_to_dict()`/`_from_dict()` round-trips `created_at`; defaults to now for legacy JSON without it
- Updated `get_due()` to iterate croniter from `created_at` anchor: finds first interval > last_run, fires only if <= now
- No catch-up after downtime: only the current interval fires, missed intervals are skipped
- Updated 3 existing tests to use explicit `created_at` (required since `Schedule.create()` now uses real clock)
- Added 3 new tests: no-catch-up after downtime, same-interval skip, next-interval fire
- Files: `awfulclaw/scheduler.py`, `tests/test_scheduler.py`
- **Learnings for future iterations:**
  - Tests that use `Schedule.create()` and check `get_due()` against a fixed `now` must set `created_at` explicitly, otherwise `created_at` is in the future relative to the test's `now` and no cron has elapsed
  - Safety bound (10000 iterations) on croniter loop prevents infinite loop for pathological cron expressions
